Creating New Lessons

Author

SSG Forensics Team

Published

November 14, 2025

WarningTechnical Users Only

This guide is intended for developers and technical users who are comfortable with HTML, Quarto, and web development. If you’re looking for user guides on how to use the application, please see the Image Gallery and Upload Guide instead.

Overview

This guide explains how to create new forensic science lessons for the SSG High School Forensics Application. Lessons are built using Quarto (QMD files) and can include text, images, videos, interactive quizzes, and custom styling.

Lesson Structure

Directory Organization

Each lesson is contained in its own directory within forensics-ui/. The structure follows this pattern:

forensics-ui/
├── lesson-1/           # Example lesson
│   ├── index.qmd       # Lesson landing page
│   ├── 1-objectives.qmd
│   ├── 2-saliva.qmd
│   ├── 3-skin.qmd
│   ├── 4-blood.qmd
│   ├── 5-buccal.qmd
│   ├── 6-epithelial.qmd
│   ├── 7-quiz.qmd
│   ├── 8-analysis-introduction.qmd
│   ├── styles.css      # Lesson-specific styles
│   └── assets/         # Images, videos, etc.
└── lesson-2/           # Your new lesson
    ├── index.qmd
    ├── 1-topic.qmd
    ├── 2-topic.qmd
    └── ...

Creating a New Lesson

Step 1: Create Lesson Directory

  1. Navigate to forensics-ui/
  2. Create a new directory with a descriptive name (e.g., lesson-2, dna-analysis, etc.)
cd forensics-ui
mkdir lesson-2
cd lesson-2

Step 2: Create the Index Page

Create an index.qmd file as the lesson landing page:

---
title: "Lesson 2: DNA Analysis"
---

## Introduction

Brief overview of what this lesson covers.

## Topics Covered

- [Topic 1](1-topic1.qmd)
- [Topic 2](2-topic2.qmd)
- [Quiz](3-quiz.qmd)

Step 3: Create Lesson Pages

Each lesson page should be a separate QMD file. Number them sequentially for organization.

Example: 1-introduction.qmd

---
title: "Introduction to DNA"
---

## What is DNA?

Your content here...

### Key Concepts

- Concept 1
- Concept 2

## Next Steps

[Continue to next section →](2-structure.qmd)

Step 4: Add Custom Styling (Optional)

Create a styles.css file for lesson-specific styling:

/* Lesson 2 Custom Styles */
.lesson-header {
  background-color: #4a90e2;
  padding: 2rem;
  color: white;
}

.key-term {
  font-weight: bold;
  color: #2c3e50;
}

Reference it in your QMD files:

---
title: "Page Title"
format:
  html:
    css: styles.css
---

Step 5: Add to Site Navigation

Edit forensics-ui/_quarto.yml to add your lesson to the site navigation:

website:
  navbar:
    left:
      - text: Home
        href: index.qmd
      - text: Lesson 1
        href: lesson-1/index.qmd
      - text: Lesson 2        # Add this
        href: lesson-2/index.qmd
      - text: Gallery
        href: cell-gallery/gallery.qmd

Step 6: Render and Test

cd forensics-ui
quarto render

Open the generated site in _site/ to test your lesson.

Best Practices

Content Organization

  1. Break content into manageable chunks: Each page should cover one main topic
  2. Use descriptive filenames: 3-dna-extraction.qmd is better than page3.qmd
  3. Number pages sequentially: Makes navigation and ordering clear
  4. Include navigation links: Add “Previous” and “Next” links at the bottom of each page

Styling

  1. Keep styles consistent: Use existing lesson styles as a template
  2. Use semantic HTML: Use <article>, <section>, etc. appropriately
  3. Make it responsive: Test on different screen sizes
  4. Follow accessibility guidelines: Use alt text, proper heading hierarchy, etc.

Media Files

  1. Optimize images: Compress images before adding them
  2. Use web-friendly formats: JPG for photos, PNG for graphics, WebP when possible
  3. Provide alt text: Always include descriptive alt text for images
  4. Host large videos externally: Use YouTube or Vimeo for large video files

Navigation

  1. Provide clear navigation: Table of contents, breadcrumbs, next/previous links
  2. Link back to lesson index: Always provide a way to return to the lesson overview
  3. Use descriptive link text: “Learn about DNA structure” instead of “Click here”

## Introduction

DNA (Deoxyribonucleic Acid) is the hereditary material in humans and most other organisms.

## The Double Helix

![DNA Double Helix](assets/dna-helix.png){width=50%}

The structure of DNA was discovered by Watson and Crick in 1953.

### Key Components

1. **Sugar-phosphate backbone**
2. **Nitrogenous bases**
   - Adenine (A)
   - Thymine (T)
   - Guanine (G)
   - Cytosine (C)

Testing Your Lesson

Local Testing

  1. Render the lesson:

    cd forensics-ui
quarto render lesson-2/

  2. Check the output in forensics-ui/_site/lesson-2/

  3. Open in a browser to test:

    # Start a local server
python -m http.server 8080 --directory _site

  4. Navigate to http://localhost:8080/lesson-2/

Checklist

Deploying Your Lesson

Once your lesson is complete and tested:

  1. Ensure all files are committed to version control

  2. Render the entire site:

    cd forensics-ui
quarto render

  3. The Docker setup will serve the new lesson automatically

Troubleshooting

Lesson Not Appearing in Navigation

  • Check that you added it to _quarto.yml
  • Verify the file path is correct
  • Re-render the entire site

Styles Not Applying

  • Ensure css: styles.css is in the YAML front matter
  • Check that the CSS file is in the correct directory
  • Clear browser cache and refresh

Images Not Loading

  • Verify image paths are correct (relative to the QMD file)
  • Check that images are in the assets/ folder
  • Ensure image file names match exactly (case-sensitive)

Videos Not Playing

  • Test video format compatibility (MP4 is most widely supported)
  • Check video file size (large files may need external hosting)
  • Verify video codec is web-compatible
